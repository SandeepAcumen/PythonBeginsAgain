Q3: What is indexing? Types?

Indexing is a database optimization technique used to speed up data retrieval from a table.
1.Clustered Index:
A Clustered Index determines the physical order of data in a table.
  a. Data rows are stored in sorted order based on the clustered key.
  b. Only ONE clustered index per table (because data can be sorted only one way).
  SQL: create clustered_index idx_empid
       on Employees(Employee_id)

2.Non-Clustered Index:
A Non-Clustered Index creates a separate structure that stores:
  a.Indexed column value
  b.Pointer to actual data row
  c.You can have multiple non-clustered indexes per table
  SQL: create nonclustered index inx_name
       on Employee(emp_id);

3.Composite Index:
An index created on Multiple columns
   SQL:Create index idx_city
   on Customers(name,city);

4.covering Index:
An index that contains all the columns required by a query, so the database 
doesn’t need to access the actual table.
   SQL: Create index idx_cover
        on orders(customer_id)
        include (orderDate,Amount);

Q10: What is Integration Runtime?

Integration Runtime (IR) is the compute infrastructure used by ADF to move and transform data.
It is the engine that actually executes activities in a pipeline.

a.Azure Integration Runtime
        Used For:
            Cloud-to-cloud data movement
            Running Mapping Data Flows

        Features:
            Fully managed by Azure
            No setup required
            Auto scaling
        Example:
            Copy from Azure SQL → ADLS

b.Self-Hosted Integration Runtime (SHIR)
        Used For:
            On-premises data sources
            Private network data sources
        Installed On:
            On-prem server
            VM inside private network
        Example:
            On-prem SQL Server → Azure Data Lake

c.Azure SSIS Integration Runtime
        Used For:
            Running SSIS packages in Azure
        For organizations migrating from:
            SQL Server Integration Services (SSIS)

Q14: What is Delta Lake?

Delta Lake is an open-source storage layer built on top of data lakes that provides 
ACID transactions, schema enforcement, time travel, and support for updates and deletes,
making data lakes more reliable and production-ready.

Where have you used Delta Lake?
👉 We used Delta tables in Azure Databricks to store transformed data in Silver and 
Gold layers. It helped us manage schema evolution, perform upserts using MERGE statements, 
and maintain data versioning.

DATA WARE HOUSE

31. What is a Fact Table?

A fact table stores transactional or measurable business data like sales amount or quantity.
 It contains foreign keys referencing dimension tables and is usually very large. It is 
 mainly used for aggregations and reporting.

Key Characteristics:
        1.Very large table
        2.Grows continuously
        3.Mostly numeric columns
        4.Used in aggregations (SUM, COUNT, AVG)

----------------------------------------------------------------------------------
----------------------------------------------------------------------------------

32. What is a Dimension Table?

A dimension table contains descriptive attributes like customer name, city, product 
category etc. It provides context to fact data and is used for filtering and grouping 
in reports.

Characteristics:
        1.Smaller than fact tables
        2.Contains descriptive text columns
        3.Used for filtering and grouping

----------------------------------------------------------------------------------
----------------------------------------------------------------------------------

33. What is Star Schema?

Star schema is a data warehouse design where a central fact table connects directly
to multiple dimension tables. It improves query performance and simplifies reporting.

Why Used?
    1.Simple design
    2.Faster queries
    3.Easy to understand

----------------------------------------------------------------------------------
----------------------------------------------------------------------------------

34. What is Snowflake Schema?

Snowflake schema is an extension of star schema where dimension tables are normalized 
into multiple related tables to reduce redundancy.

Dimension tables are split into sub-dimensions.

----------------------------------------------------------------------------------
----------------------------------------------------------------------------------

35. What is SCD Type 1?

Slowly Changing Dimension Type 1
SCD Type 1 overwrites existing data without maintaining history. It is simple but 
historical changes are lost.

----------------------------------------------------------------------------------
----------------------------------------------------------------------------------

36. What is SCD Type 2?

SCD Type 2 preserves historical data by inserting a new record when changes occur 
and expiring the previous record using start and end dates.

----------------------------------------------------------------------------------
----------------------------------------------------------------------------------

37. What is a Surrogate Key?

A surrogate key is a system-generated unique identifier used in dimension tables 
instead of business keys to maintain consistency and improve performance.

----------------------------------------------------------------------------------
----------------------------------------------------------------------------------

38. Why Do We Partition Large Tables?

Partitioning splits large tables into smaller segments based on a column like date. 
It improves query performance and reduces scan time.

Purpose:
    1.Improve query performance
    2.Manage large datasets
    3.Faster loading
    4.Easier maintenance

39. What is Data Skew?

Data skew occurs when data is unevenly distributed across partitions, causing 
performance bottlenecks in distributed systems.

Solution:
    Repartition
    Use salting
    Choose better partition key

41. Why is Parquet Better than CSV?

Parquet is columnar, compressed, and schema-aware, making it more efficient for 
analytics compared to CSV.

CSV:

    Larger size
    No schema

Parquet:

    Faster queries
